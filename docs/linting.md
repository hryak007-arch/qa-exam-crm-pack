# Linting

## Обрані інструменти
У проєкті використано:
- **Ruff** — для перевірки стилю, типових помилок, імпортів і частини best practices;
- **mypy** — для статичної перевірки типів;
- **pre-commit** — для автоматичного запуску перевірок перед комітом.

## Причини вибору
Ruff обрано через високу швидкодію, підтримку великої кількості правил та можливість поєднати linting і formatting.
mypy використано для підвищення надійності Python-коду через перевірку типів.
pre-commit дозволяє автоматизувати контроль якості до потрапляння змін у репозиторій.

## Базові правила
- перевірка синтаксичних та логічних помилок;
- контроль імпортів;
- базові правила стилю;
- перевірка частини FastAPI-специфічних проблем;
- перевірка типів через mypy.

## Ігнорування
З перевірки виключено:
- `upload/`
- `reports/`
- `htmlcov/`
- кеш-директорії

## Запуск
```bash
pip install -r requirements-dev.txt
python -m ruff check .
python -m ruff format --check .
python -m mypy .
