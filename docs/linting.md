# Linting

## Обрані інструменти
У проєкті використано:
- **Ruff** — для перевірки стилю, помилок, імпортів та базових best practices;
- **mypy** — для статичної перевірки типів;
- **pre-commit** — для автоматичного запуску перевірок перед комітом.

## Причини вибору
Ruff обрано через високу швидкодію та поєднання linting і formatting.
mypy використано для підвищення надійності Python-коду через перевірку типів.
pre-commit дозволяє автоматизувати контроль якості до потрапляння змін у репозиторій.

## Базові правила
- перевірка синтаксичних та логічних помилок;
- контроль імпортів;
- базові правила стилю;
- перевірка типів через mypy.

## Ігнорування
З перевірки виключено:
- `upload/`
- `reports/`
- `htmlcov/`
- кеш-директорії

## Запуск
```bash
pip install -r requirements-dev.txt
py -m ruff check .
py -m ruff format --check .
py -m mypy .
```

## Git hooks
```bash
py -m pre_commit install
py -m pre_commit run --all-files
```

## Інтеграція з процесом збірки
Для комплексної перевірки використовується команда:
```bash
py scripts/check_quality.py
```

## Статична типізація
У проєкті використовується `mypy` для базової перевірки типів Python-коду.
